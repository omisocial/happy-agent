# Proposal: HappyAgent Self-Healing Automation
Generated by: cm-brainstorm-idea
Date: 2026-04-02

## Why we're doing this
### Qualified Problem
**For:** Boxme Operations Team / Affiliates
**Who:** Cần gửi lượng lớn lời mời hợp tác cho các Creator trên TikTok Shop Partner Center, nhưng quá trình thao tác tay tốn nhiều thời gian và dễ sai sót.
**The:** Các tool tự động hóa hiện tại (như Puppeteer/Playwright thuần) là một giải pháp
**That:** Rất dễ bị "gãy" (break) khi TikTok thay đổi giao diện, class name, hoặc cấu trúc DOM. Mỗi lần gãy, dev phải vào đọc log, tìm DOM mới, sửa code và deploy lại, làm gián đoạn vận hành và tốn chi phí maintain.
**Unlike:** Giải pháp cứng nhắc dựa trên auto-generated CSS selectors.
**Our approach:** Xây dựng một hệ thống "Self-healing" (tự phục hồi). Kết hợp tốc độ của Automation script (Node.js + Playwright) với khả năng linh hoạt và suy luận của AI (Healer Agent) để tự động đọc DOM lúc lỗi, tìm lại selector đúng, sửa code và tự động tiếp tục chạy mà không cần sự can thiệp của con người.

## What's changing
### Options Evaluated
1. **Option A: Pure AI Browser Agent (e.g., Use Claude to browse every page)**
   - *Approach:* Giao toàn quyền cho AI điều khiển trình duyệt từng thao tác.
   - *Pros:* Cực kỳ linh hoạt, code ngắn.
   - *Cons:* Rất chậm (mỗi bước tốn chục giây chờ AI), tốn kém chi phí API cực lớn do phải stream DOM liên tục.
2. **Option B: Self-Healing Hybrid Architecture (Đề xuất)**
   - *Approach:* Chạy bằng script tĩnh (Playwright), khi nào lỗi mới gọi AI vào sửa config.
   - *Pros:* Nhanh như tool bot bình thường, zero-cost cho các lượt chạy chuẩn, chỉ tốn token khi có lỗi (và sửa xong dùng lại cho các lượt sau).
   - *Cons:* Cần kiến trúc code phức tạp hơn (Runner <-> Tracer <-> Healer <-> Bash loop).

### Recommendation
**Choose Option B: Self-Healing Hybrid Architecture**
- Tối ưu chi phí và tốc độ vận hành vì 99% thời gian bot chạy bằng Node.js thuần.
- AI chỉ can thiệp vào `1%` sự cố, hoạt động như một "kỹ sư trực page" tự động sửa code.
